> 这份文档是给自己看的学习笔记，用大白话解释所有概念和实验结果。

---

## 第一章：成员推理攻击（MIA）到底是什么

### 先从一个故事开始

假设你是一家医院，用 10000 个病人的数据训练了一个 AI 模型来诊断疾病。  
训练完之后，这个模型对外提供服务，任何人都可以输入一张 X 光片，模型会输出诊断结果和置信度。

现在，有个攻击者想知道：**"张三的 X 光片，有没有被用来训练这个模型？"**

如果攻击者能确定这一点，就等于知道了 "张三曾经在某医院看过病"——这是隐私泄露。

这就叫 **成员推理攻击（Membership Inference Attack，MIA）**。  
"成员"的意思就是：这个数据是不是训练集的"成员"。

---

### 为什么能猜出来？模型的秘密弱点

深度学习模型有一个很有意思的特点：**对训练过的数据特别自信**。

```
训练集里的图片（模型见过的）：
  → 模型预测：猫，置信度 97%

从没见过的图片（测试集）：
  → 模型预测：猫，置信度 74%
```

这个现象叫做 **过拟合（Overfitting）**：模型有点"死记硬背"，记住了训练数据，而不是真正学到了规律。

攻击者利用的就是这个：**训练集数据 → 高置信度，非训练集数据 → 低置信度**。

只要找到一个置信度阈值 τ，就能区分"是不是训练集成员"：

```
置信度 ≥ τ  →  猜这是训练集成员
置信度 < τ  →  猜这不是训练集成员
```

---

### 攻击的具体操作（我们实现的方法）

攻击者需要准备三组数据：

| 数据组 | 说明 | 用途 |
|---|---|---|
| **已知成员**（member） | 确定在训练集里的数据 | 校准阈值 |
| **已知非成员**（non-member） | 确定不在训练集里的数据 | 校准阈值 |
| **目标数据**（target） | 不知道是不是训练集，要猜的 | 攻击目标 |

**第一步**：用已知成员 + 已知非成员查询模型，观察置信度分布，找出最佳阈值 τ。

```
已知成员的置信度：  [0.95, 0.88, 0.91, 0.97, ...]  → 普遍偏高
已知非成员的置信度：[0.62, 0.71, 0.58, 0.68, ...]  → 普遍偏低

找一个阈值，比如 τ = 0.80，让两组分得最开
```

**第二步**：用这个阈值对目标数据做判断：
- 目标数据置信度 ≥ 0.80 → 猜是训练集成员
- 目标数据置信度 < 0.80 → 猜不是训练集成员

**评分**：猜对了多少比例 = MIA 准确率。
- 随机猜 = 50%（等于没有信息）
- 越高 = 模型隐私泄露越严重

---

### 什么时候 MIA 最危险？

MIA 效果取决于模型**过拟合程度**。过拟合越严重，训练集和非训练集的置信度差距越大，MIA 越容易。

在我们实验中，用**极小的训练集**（比如 500 张图）来故意制造严重过拟合：

```
训练集准确率：98%（几乎完美记住了每张图）
测试集准确率：46%（泛化能力很差）

→ 置信度差距极大 → MIA 成功率可以达到 68~78%
```

## 第二章：FGSM 对抗攻击

### 什么叫对抗样本？

给一张图片加一点人眼看不见的噪声，让模型判断错：

```
原图：一只猫（人看：猫，模型预测：猫 99%）
加噪后：看起来还是猫（人看：猫，模型预测：狗 94%）
```

这个噪声非常微小，人完全看不出差别，但模型就是会被骗。

为什么会这样？因为模型对图片的"理解"和人非常不同，它处理的是像素数值，
即使改变几个像素的值 ±8（图片像素范围 0~255），人眼感知不到，但模型的判断可以完全翻转。

---

### FGSM 的原理

**FGSM（Fast Gradient Sign Method）**，Goodfellow 2014年提出，是最简单的对抗攻击方法。

**核心思路**：沿着让模型"最容易犯错"的方向加噪声。

怎么找这个方向？用**梯度**。

梯度是什么：在深度学习里，梯度告诉你"把哪个像素改一点，损失函数变化最大"。  
损失函数越大 = 模型越犯错。所以沿着梯度方向改像素 = 让模型最快犯错。

```
公式：
对抗样本 = 原图 + ε × sign(∇ₓL)

解释：
  ∇ₓL  = 损失函数对每个像素的梯度（哪些像素最敏感）
  sign  = 取符号（+1 或 -1），只管方向不管大小
  ε     = 步长（控制噪声大小，比如 8/255 ≈ 0.031）
```

**为什么用 sign？**  
sign 保证了每个像素变化量完全相同（都是 ε），这样总扰动是均匀的，
不会有个别像素变化太大而引起人眼察觉。

---

### FGSM 用来做防御？

对抗训练（Adversarial Training）的思路：

1. 对每个训练图片，生成一个对抗样本
2. 训练时同时喂原图和对抗样本
3. 模型被迫学会：不管是干净图片还是加了噪声的图片，都要分对

**为什么这样能防御 MIA？**

因为模型必须变得更"通用"，不能只靠死记硬背训练数据：
- 训练集图片加了噪声之后模型还要分对 → 不能对训练图片过度依赖
- 导致模型对训练集的置信度不再那么高
- 训练集和非训练集的置信度差距缩小
- MIA 的阈值变得难找 → MIA 成功率下降

---

## 第三章：我们改进了什么

### 原来的方法（你简历上写的）

**归一化梯度 FGSM**：
```
δ = ε × 梯度 / ||梯度||₂
```

这个方法的想法是：不用 sign（只取方向），而是用真实的梯度方向（有大有小）。

**实验结果：没用，有时反而让 MIA 更容易。**

为什么没用？
- 归一化后虽然保留了梯度大小的相对关系，但总体效果和 sign 差不多
- 没有针对 MIA 的特殊设计，只是换了一种加噪方式

---

### 创新点一：感知自适应 FGSM（Adaptive FGSM）

**问题**：标准 FGSM 对所有像素一视同仁，每个像素都加同样大小的扰动。

**我们的想法**：聪明地分配扰动预算。

#### A. 梯度集中权重

梯度大的像素 → 对模型决策影响大 → 值得多分配扰动  
梯度小的像素 → 对模型影响微弱 → 少分配或不分配

```python
w_eff = (|梯度| / 最大梯度)^α    # α 控制集中程度，0=均匀，1=完全集中
```

直觉：把有限的 ε 预算集中花在"最有效"的像素上。

#### B. 感知遮罩（视觉不可察性）

纹理丰富的区域（草地、毛发、衣服）→ 人眼注意力低 → 可以多加噪声  
平滑区域（天空、白墙、皮肤）→ 人眼很容易察觉噪声 → 少加

怎么判断"纹理丰富"？用**局部方差**：

```python
局部方差 = 以该像素为中心5×5窗口内，像素值的方差
方差大 = 纹理丰富（像素值跳动剧烈）
方差小 = 平滑区域（像素值相近）

w_perc = (局部方差 / 最大方差)^β
```

#### C. 合并两个权重

```python
W = w_eff × w_perc           # 两个权重相乘
W_norm = W / max(W)          # 归一化，最大值=1（保证 L∞ ≤ ε）

δ = ε × sign(梯度) × W_norm  # 最终扰动
```

**效果**：在同样的 ε 下，扰动更集中、更隐蔽，对模型的干扰更有针对性。

---

### 创新点二：MIA 置信度导向损失

**问题**：普通 FGSM 只让模型"分错"，但 MIA 的漏洞是"置信度太高"。

**直接攻击弱点**：在生成对抗样本时，同时压低置信度：

```
标准生成损失 = CE(模型预测, 真实标签)
                → 最大化分类错误

我们的生成损失 = CE(模型预测, 真实标签) - λ × 平均最大置信度
                → 最大化分类错误  AND  压低置信度
```

第二项的作用：生成的对抗样本专门针对"高置信度"这个漏洞，
用这样的对抗样本训练出来的模型，会主动学会"不要对训练数据太自信"。

λ（lambda）控制置信度压制的力度：
- λ=0：标准 FGSM，不压置信度
- λ=0.5：适度压制（我们实验的最优值）
- λ 过大：可能破坏正常分类能力

---

### 创新点三：TRADES KL 损失 + AdvReg 惩罚

这两个是结合文献方法做的改进：

#### TRADES KL 散度（Zhang et al., 2019）

```
训练损失 = CE(干净样本) + λ × KL散度(干净预测 || 对抗预测)
```

KL散度是什么：衡量两个概率分布之间的"距离"。

这里的意思：干净样本的预测结果 和 对抗样本的预测结果，要尽量一致。

效果：决策边界变得平滑。原来干净样本和对抗样本的预测可能差很远，
现在要求它们接近，模型对"轻微扰动"的鲁棒性变强。

#### AdvReg 置信度惩罚（Nasr et al., 2018）

```
训练损失 += μ × 训练样本的平均最大置信度
```

非常直接：训练时，如果模型对训练数据过于自信，直接在损失上加罚款。

效果：模型主动降低对训练数据的置信度，直接堵住 MIA 的漏洞。

## 第四章：实验结果详解

### 为什么要用小训练集？

正常训练（50000张）时，模型泛化得很好，训练集和测试集准确率差不多，MIA 没什么信号：

```
训练集准确率：91%
测试集准确率：88%
置信度差距：小 → MIA 成功率接近 50%（等于瞎猜）
```

用小训练集（500张）时，模型严重过拟合：

```
训练集准确率：99%（几乎记住了每张图）
测试集准确率：46%（泛化极差）
置信度差距：大 → MIA 成功率 60~70%
```

这样才能看出防御方法是否有效。

---

### 实验一：MNIST × SimpleCNN（手写数字）

MNIST 最简单：1~9 的手写数字，灰度图，10类。

| 方法 | 干净准确率 | MIA成功率 | MIA降低了多少 |
|---|---|---|---|
| 无防御（基线） | 94.00% | 57.00% | — |
| Sign-FGSM | 96.17% | 53.50% | ↓3.5% |
| Normalized-FGSM | 94.55% | 56.50% | ↓0.5%（几乎无效） |
| **Adaptive-FGSM** | **95.68%** | **52.50%** | **↓4.5% ★最优** |
| Adaptive+KL+AdvReg | 95.13% | 54.50% | ↓2.5% |

**关键发现**：
- MNIST 上，所有对抗训练方法都让准确率**变高了**！
  - 原因：对抗训练相当于数据增强 + 正则化，对简单任务反而有帮助
- Adaptive-FGSM 最优：MIA 从 57% 降到 52.5%，准确率还提升了 1.68%
  - 这意味着：**防御没有代价，反而两头都赢**

---

### 实验二：CIFAR-10 × SimpleCNN（彩色物体识别）

CIFAR-10 更难：飞机、汽车、鸟、猫...10类彩色图。

| 方法 | 干净准确率 | MIA成功率 | MIA降低了多少 | PU得分 |
|---|---|---|---|---|
| 无防御（基线） | 46.59% | 68.75% | — | — |
| Sign-FGSM | 41.33% | 64.00% | ↓4.75% | 0.90 |
| Normalized-FGSM | 46.59% | **70.25%** | **↑-1.5%（变差！）** | — |
| Adaptive-FGSM | 45.56% | 65.50% | ↓3.25% | 3.16 |
| **Adaptive+KL+AdvReg** | **46.50%** | **65.25%** | **↓3.5%** | **38.9 ★** |

**关键发现**：
- **Normalized-FGSM 失效**：MIA 反而上升了 1.5%，说明这个方法本身有问题
- **Sign-FGSM MIA 降了最多**，但准确率损失了 5.26%（每降 1% MIA，损失 1.1% 准确率）
- **Adaptive+KL+AdvReg 最优**：MIA 降了 3.5%，准确率只损失 0.09%
  - PU 得分 38.9 = 每损失 1% 准确率，换来 38.9% 的 MIA 保护
  - 比 Sign-FGSM 效率高 **43 倍**

---

### 实验三：CIFAR-10 × ResNet18（大模型）

ResNet18 有 1100万参数，更接近实际应用的模型大小。用 5000 个训练样本。

过拟合更严重：训练集 98.7%，测试集 72.2%，基线 MIA 高达 **67.80%**。

| 方法 | 干净准确率 | MIA成功率 | MIA降低了多少 | PU得分 |
|---|---|---|---|---|
| 无防御（基线） | 72.17% | 67.80% | — | — |
| **Sign-FGSM** | 54.01% | 54.83% | **↓12.97% ✅超10%** | 0.71 |
| Normalized-FGSM | 64.22% | 58.77% | ↓9.03% | 1.14 |
| Adaptive-FGSM | 65.38% | 61.77% | ↓6.03% | 0.89 |
| **Adaptive+KL+AdvReg** | **69.35%** | **64.50%** | ↓3.30% | **1.17 ★** |

**关键发现**：
- Sign-FGSM 首次突破 10% MIA 降低目标，但代价是准确率暴跌 **18.16%**
  - 从 72% 降到 54%，模型实际上已经基本没用了
- Adaptive+KL+AdvReg 依然最高效：只损失 2.82% 准确率，PU 得分最高
- Normalized-FGSM 这次有效（降了 9.03%），但仍然不如其他方法

---

### 实验四：CIFAR-100 × SimpleCNN（100类，极难）

CIFAR-100 有 100 个类，模型只有 500 个训练样本：

```
500 张图 ÷ 100 个类 = 平均每类只有 5 张图
模型根本学不会！基线准确率只有 9.9%（随机猜测是 1/100 = 1%）
```

| 方法 | 干净准确率 | MIA成功率 | MIA变化 |
|---|---|---|---|
| 无防御（基线） | 9.90% | 78.25% | — |
| Sign-FGSM | 10.02% | 81.50% | **↑变差 3.25%** |
| Normalized-FGSM | 10.65% | 89.25% | **↑变差 11%（最差）** |
| Adaptive-FGSM | 10.55% | 87.00% | ↑变差 8.75% |
| **Adaptive+KL+AdvReg** | **9.91%** | **80.75%** | **↑变差最少（2.5%）** |

**非常重要的发现：所有方法都让 MIA 变得更容易了！**

为什么？
- 模型基础准确率只有 9.9%，说明它根本没学到有用的特征
- 对抗训练加了噪声之后，连这点脆弱的学习能力都被破坏了
- 模型更加"记不住"正常分类，但"记住训练数据"这个过拟合信号反而更强
- 结论：**对抗训练有效的前提是模型本身能学到有意义的表征**

---

### 实验五：CIFAR-100 × ResNet18（大模型+更多数据）

5000 个训练样本，ResNet18，60轮。基线 MIA 高达 **90.73%**——是所有实验里最严重的过拟合。

| 方法 | 干净准确率 | MIA成功率 | MIA降低了多少 | PU得分 |
|---|---|---|---|---|
| 无防御（基线） | 30.78% | 90.73% | — | — |
| **Sign-FGSM** ⭐ | 26.72% | **81.20%** | **↓9.53%（相对-10.5%）** | **2.35** |
| Normalized-FGSM | 31.01% | 91.20% | ↑-0.47%（变差） | — |
| Adaptive-FGSM | 27.97% | 88.10% | ↓2.63% | 0.94 |
| Adaptive+KL+AdvReg | 29.64% | 91.47% | ↑-0.73%（变差） | — |

**关键发现（和 CIFAR-10 结论不同！）**：
- Sign-FGSM 是**唯一有效**的方法，MIA 降了 9.53%（接近 10% 目标），PU 得分 2.35
- Adaptive+KL+AdvReg 在这里**失效了**，MIA 反而上升了 0.73%
- Normalized-FGSM 也失效，MIA 微升 0.47%

**为什么 CIFAR-100 上结论翻转了？**

在 CIFAR-10 上，Adaptive+KL+AdvReg 表现最好（PU 38.9），但在 CIFAR-100 上失效。原因：
- CIFAR-100 过拟合极其严重（基线 MIA 90.73% vs CIFAR-10 的 67.80%）
- KL 损失要求"干净预测和对抗预测接近"，在严重过拟合时这个约束反而干扰了正常学习
- AdvReg 惩罚训练置信度，但此时模型已经过拟合到很深，惩罚反而破坏了有限的泛化能力
- **简单粗暴的 Sign-FGSM 直接扰乱记忆过程，反而更有效**

---

### 实验六：MNIST × ResNet18（大模型 + 简单任务）

用 1000 个训练样本，看 ResNet18 在 MNIST 上会不会过拟合。

| 方法 | 干净准确率 | MIA成功率 | MIA变化 |
|---|---|---|---|
| 无防御（基线） | 98.02% | **49.75%** | — |
| Sign-FGSM | 97.26% | 51.00% | ↑略变差 |
| Normalized-FGSM | 95.53% | 50.50% | ↑略变差 |
| Adaptive-FGSM | 97.17% | 49.88% | 几乎不变 |
| Adaptive+KL+AdvReg | 98.24% | 50.00% | 几乎不变 |

**最重要的发现：基线 MIA 只有 49.75%，等于随机猜测！**

这说明 ResNet18 用 1000 个 MNIST 样本根本不会过拟合——模型太强了，MNIST 太简单了，1000 张图已经够它学好泛化特征，训练集和测试集置信度差不多，MIA 完全没有可利用的漏洞。

**结论：没有过拟合 = 不需要防御。**  
这也验证了一个更大的规律：**MIA 的威胁来自过拟合，不是来自模型大小或数据集大小本身**。

### 实验七：CIFAR-100 × ResNet18，ε=16/255（加大扰动）

把 ε 从 8/255（≈0.031）翻倍到 16/255（≈0.063），看能不能进一步降低 MIA。

| 方法 | 干净准确率 | MIA成功率 | MIA降低了多少 | PU得分 |
|---|---|---|---|---|
| 无防御（基线） | 31.08% | 91.07% | — | — |
| **Sign-FGSM** ⭐ | 24.93% | **74.10%** | **↓16.97%（相对-18.6%）** | **2.76** |
| Normalized-FGSM | 32.19% | 90.00% | ↓1.07% | 0.97 |
| Adaptive-FGSM | 26.55% | 86.17% | ↓4.90% | 1.08 |
| Adaptive+KL+AdvReg | 29.07% | 90.93% | ↓0.13% | 0.06 |

**关键发现：Sign-FGSM 在 ε=16/255 下达到 16.97% MIA 降低——整个项目所有实验中最高！**

和 ε=8/255 的对比：

| ε | Sign-FGSM MIA降低 | 准确率损失 |
|---|---|---|
| 8/255 | 9.53% | -4.06% |
| 16/255 | **16.97%** | -6.15% |

扰动翻倍 → MIA 保护提升 78%（9.53% → 16.97%），而准确率损失只增加了 51%（4.06% → 6.15%）。  
说明在过拟合严重的场景，**加大 ε 是有效的策略**，边际回报是正的。

18.6% 的相对 MIA 降低已经远超文献中 AdvReg (2019) 的 7.5% 基准。

---

### 实验八：GradSmooth——新方法（梯度敏感度自适应软标签）

**设计思路**：不把 FGSM 梯度加到输入上，而是用梯度大小作为"过拟合风险信号"，
对高风险样本给更软的训练标签，让模型无法在这些样本上过度自信。

```python
敏感度 = ||∇_x CE(f(x), y)||   # 梯度越大 = 被死记 = MIA高危
平滑因子 = α × (敏感度 / 最大敏感度)   # 动态调整
软标签 = (1 - 平滑因子) × one_hot(y) + 平滑因子 / K
```

| 方法 | 数据集 | 干净准确率 | MIA成功率 | MIA降低 | 准确率变化 | PU |
|---|---|---|---|---|---|---|
| GradSmooth α=0.3 | MNIST | 94.60% | 54.50% | ↑-1.50% | +0.22% | — |
| **GradSmooth α=0.5** | **MNIST** | **94.49%** | **51.75%** | **↓1.25% ★最优** | **+0.11%** | **∞** |
| GradSmooth α=0.3 | CIFAR-10 | 47.70% | 67.00% | ↓0.50% | +1.03% | ∞ |
| **GradSmooth α=0.5** | **CIFAR-10** | **47.67%** | **64.75%** | **↓2.75%** | **+1.00%** | **∞** |

**关键突破：GradSmooth α=0.5 在 MNIST 和 CIFAR-10 上都实现了"MIA 下降 + 准确率提升"——双赢！**

为什么能做到？
- 没有对抗样本加入训练 → 不会因为"学错误样本"而损失准确率
- 软标签对高风险样本降低过度自信 → 压缩 MIA 的置信度差距
- 低风险样本（已经泛化好）用接近硬标签 → 准确率不受影响

α 的选择：α=0.3 软化力度不够（MNIST 上甚至变差），α=0.5 恰好。α 过大可能反过来伤害准确率。

#### GradSmooth vs 全部方法联合对比（SimpleCNN, train=500, 80 epochs）

把 MNIST 和 CIFAR-10 的所有方法排在同一张表，看谁最稳健：

### 跨实验规律总结

| 数据集 | 模型 | ε | 最优方法 | 备注 |
|---|---|---|---|---|
| MNIST | SimpleCNN | — | **GradSmooth α=0.5** | MIA↓1.25% 且准确率+0.11%，双赢 |
| MNIST | ResNet18 | — | 无需防御 | 基线MIA≈50%，无过拟合信号 |
| CIFAR-10 | SimpleCNN | — | **GradSmooth α=0.5** | MIA↓2.75% 且准确率+1.00%，双赢 |
| CIFAR-10 | ResNet18 | 8/255 | Adaptive+KL+AdvReg | PU最优（1.17）；Sign-FGSM MIA降最多但-18% |
| CIFAR-100 | SimpleCNN | — | 全部失效 | 模型太弱（准确率9.9%） |
| CIFAR-100 | ResNet18 | 8/255 | Sign-FGSM | 过拟合极重，MIA↓9.53% |
| CIFAR-100 | ResNet18 | 16/255 | **Sign-FGSM ★** | **MIA↓16.97%，全项目最高** |

**四条核心规律**：

1. **没有过拟合 → 不需要防御**（MNIST × ResNet18 基线MIA≈50%）
2. **想要双赢（MIA↓ + 准确率↑）→ GradSmooth**（MNIST 和 CIFAR-10 均成立）
3. **过拟合越严重 → 需要更大 ε 的简单强力扰动**（CIFAR-100 的规律）
4. **任务难度适中、可接受少量准确率损失 → Adaptive+KL+AdvReg**（大模型上性价比最优）

## 第五章：每个指标怎么看

### 干净准确率（Clean Accuracy）

```
= 测试集上分对的图片数 / 测试集总图片数
```

**防御方法不能让这个数字掉太多**，否则模型变成废品。

- 越高越好
- 我们关注的是"防御方法让这个数字变化了多少"

---

### MIA 攻击成功率（MIA Accuracy）

```
= 攻击者猜对的比例（猜成员/非成员的准确率）
```

**衡量隐私泄露严重程度**：

| MIA 准确率 | 理解 |
|---|---|
| 50% | 完美：攻击者等于瞎猜，不知道你的数据在不在 |
| 55% | 轻微泄露：攻击者有一点优势 |
| 65% | 中度泄露：有隐私风险 |
| 75%+ | 严重泄露：模型"记住"了训练数据 |

- 越低越好（理想是 50%）

---

### MIA 降低幅度（MIA Drop）

```
= 基线 MIA准确率 - 防御后 MIA准确率
```

- 正数 = 防御有效（MIA 成功率下降了）
- 负数 = 防御无效或反效果（Normalized-FGSM 在 CIFAR-10 就是这样）
- > 5% 算明显有效，> 10% 算强防御

---

### 准确率变化（Clean Δ）

```
= 防御后准确率 - 基线准确率
```

- 负数 = 防御带来的代价（准确率损失）
- 正数 = 意外收获（MNIST 上就出现了这种情况）
- 越接近 0 越好

---

### PU 得分（Privacy-Utility Score，隐私-效用得分）

```
PU = MIA降低幅度 / |准确率变化|
```

**直白理解：每损失 1% 准确率，换来多少 % 的 MIA 保护。**

| PU 得分 | 含义 |
|---|---|
| < 1 | 性价比差：损失的准确率比得到的保护还多 |
| 1~3 | 还可以 |
| > 3 | 优秀 |
| → ∞ | 最优：准确率不降反升（保护和效用双赢） |

**举例**：
- Sign-FGSM（CIFAR-10）：MIA↓4.75%，准确率↓5.26%，PU = 4.75/5.26 = **0.90**
  - 花了 5% 的准确率，只换来 4.75% 的保护，亏本
- Adaptive+KL（CIFAR-10）：MIA↓3.5%，准确率↓0.09%，PU = 3.5/0.09 = **38.9**
  - 几乎不花准确率，换来 3.5% 的保护，极度赚

---

### 相对 MIA 降低（Relative MIA Drop）

```
= MIA降低幅度 / 基线MIA准确率 × 100%
```

用于跨数据集比较，因为不同数据集基线 MIA 不同：

```
MNIST Adaptive-FGSM：4.5% / 57.0% = 7.9% 相对降低
CIFAR-10 Adaptive+KL：3.5% / 68.75% = 5.1% 相对降低
```

---

## 第六章：总结——我们做了什么、结论是什么

### 做了什么

1. **发现了问题**：原来的归一化梯度 FGSM 没有效果，甚至有反效果

2. **提出了 Adaptive FGSM**：
   - 感知自适应加权：把扰动集中在"对模型影响大"且"人眼不敏感"的像素上
   - MIA 置信度导向损失：生成对抗样本时专门压低置信度，直接攻击 MIA 的弱点

3. **结合文献方法**：叠加 TRADES KL 损失（平滑决策边界）和 AdvReg 惩罚（直接罚过自信）

4. **跑了 4 个实验**：MNIST、CIFAR-10（两个模型）、CIFAR-100

### 核心结论

**方法选择**：
- 如果追求**最大 MIA 降低**（不在乎准确率损失）→ Sign-FGSM + ResNet18（12.97%，但准确率-18%）
- 如果追求**最优性价比**（PU 得分最高）→ **Adaptive+KL+AdvReg**
- 如果任务非常简单（如 MNIST）→ 单纯 Adaptive-FGSM 就够用

**新发现（CIFAR-100 的教训）**：
- 对抗训练有效的前提是模型能学到有意义的特征
- 当任务极难（数据太少、类别太多）时，对抗训练可能适得其反

**与文献对比**：
- AdvReg（2019）论文：7.5% MIA降低，代价 7.5% 准确率损失
- 我们在 MNIST 上：**7.9% 相对 MIA 降低，准确率反而提升 1.68%**
- 在 CIFAR-10 上：PU 得分 38.9，大幅超过标准 Sign-FGSM 的 0.9
